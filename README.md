# mynixos

NixOS flake configuring two hosts (`thinkpad`, `formd`) with niri + DankMaterialShell. GUI apps are installed declaratively via Flatpak; CLI tooling and system services come from nixpkgs.

Tracks NixOS 25.11 stable.

## Layout

```
flake.nix                       — inputs and per-host outputs
lib/mkHost.nix                  — wires nixpkgs + home-manager + 3rd-party modules + the host config
hosts/<hostname>/
  default.nix                   — picks bundles for this host
  home.nix                      — per-host home config (mainly monitor outputs)
  hardware-configuration.nix    — generated by nixos-generate-config
modules/
  nixos/                        — base system modules (boot, audio, niri, flatpak, …) — always on
  home/                         — user-facing config: shell, terminal, niri config, DMS settings
  bundles/                      — toggleable groups: each enables flatpaks + nixpkgs + system options
```

## Bundles

Toggle from a host's `default.nix`:

```nix
my.bundles = {
  dev.enable = true;
  media.enable = true;
  communication.enable = true;
  productivity.enable = true;
  browsers.enable = true;
  vpn.enable = true;
  gaming.enable = false;
};
```

A bundle can pull in flatpaks, nixpkgs CLI tools, and additional NixOS modules — see `modules/bundles/dev.nix` for the canonical example.

## Rebuild

```sh
./rebuild.sh                # rebuilds for `$(hostname)`
./rebuild.sh thinkpad       # explicit host
```

## First install on a new host

1. Install NixOS minimal, partition, set hostname to `thinkpad` or `formd`.
2. `nixos-generate-config --show-hardware-config > hosts/<host>/hardware-configuration.nix`.
3. Clone this repo into `~/repos/github.com/bulkashmak/mynixos`.
4. `sudo nixos-rebuild switch --flake .#<host>`.
5. After first boot into niri, run `niri msg outputs` to get the real monitor names; update `hosts/<host>/home.nix`.
